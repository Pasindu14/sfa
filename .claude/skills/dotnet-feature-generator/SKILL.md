# .NET API Feature Generator

Generate complete, production-ready ASP.NET Core features for the SFA API project following the exact architectural patterns established in the Users feature.

Every feature generated by this skill must be **production-ready for 1 000 concurrent field reps and scalable to 50M+ database rows without architectural rewrites**. The 14 scalability rules below are non-negotiable — they are baked into every template and must be present in every generated file.

---

## What This Skill Does

This skill automates the creation of a full-stack API feature with:
- Entity models with audit fields and soft delete via `IsActive`
- Repository pattern (interface + implementation) with `AsNoTracking`, `IsActive` filter, and `ExecuteUpdateAsync`
- Service layer with business logic and `IDistributedCache` support
- Controller with CRUD endpoints and `CancellationToken` on every action
- DTOs for responses — never raw entity returns
- Request models for inputs
- FluentValidation validators
- Dependency injection registration in Program.cs (Scoped lifetime with comments)
- DbContext registration with composite partial indexes
- EF Core migration command generation with partitioning note for high-growth entities
- Automatic build validation with iterative error fixing

## Production Scalability Rules (NON-NEGOTIABLE)

These 14 rules apply to **every file generated by this skill**. Check them before calling the feature complete.

### ALWAYS DO

| # | Rule | Pattern |
|---|------|---------|
| 1 | **Async everywhere** | `ToListAsync(ct)`, `FirstOrDefaultAsync(ct)`, `SaveChangesAsync(ct)` — every async method accepts `CancellationToken ct` |
| 2 | **Scoped lifetimes with comments** | `AddScoped<IRepo, Repo>(); // Scoped — per-request DB context` |
| 3 | **AsNoTracking on every read query** | `.AsNoTracking()` immediately after the DbSet on all list/read queries |
| 4 | **IsActive filter on every query** | `.Where(x => x.IsActive)` in every WHERE clause — `IsActive` is the universal soft-delete flag |
| 5 | **Paginate every list** | `.Skip(skip).Take(take)` — no unbounded list queries ever |
| 6 | **ExecuteUpdateAsync for status changes** | Soft delete, deactivation, status toggle — never load entity just to change one field |
| 7 | **AddRange then single SaveChangesAsync** | Batch all adds; one DB round-trip for multi-row inserts |
| 8 | **Batch WHERE IN for related data** | `.Where(u => ids.Contains(u.Id))` — never loop DB calls |
| 9 | **Composite partial indexes** | `HasIndex(new{Col,CreatedAt}).HasFilter("\"IsActive\" = true")` |
| 10 | **Project to DTO in repo or service** | `.Select(x => new Dto{...})` or `MapToDto()` — never return raw EF entities |
| 11 | **AddDbContextPool** | `services.AddDbContextPool<AppDbContext>(...)` — not `AddDbContext` |
| 12 | **Rate limiting** | Register `AddRateLimiter` with sliding-window global + auth limiters |
| 13 | **IDistributedCache only** | Use `IDistributedCache` with cache-aside pattern — never `IMemoryCache` |
| 14 | **Connection string from config** | `builder.Configuration.GetConnectionString("DefaultConnection") ?? throw ...` |

### NEVER DO

| Anti-pattern | Why it fails at scale |
|---|---|
| `.Result` or `.Wait()` on a Task | Deadlocks under ASP.NET Core synchronization context |
| `Task.Run()` wrapping sync DB calls | Wastes thread pool; doesn't make sync code async |
| `ToList()` / `Find()` / `SaveChanges()` (sync EF) | Blocks threads; disables cooperative cancellation |
| Omitting `CancellationToken ct` from async signatures | Client disconnect does not abort the DB query |
| `AddDbContext<>` without Pool | Creates/destroys a new context per request — ~20% throughput loss |
| `IMemoryCache` in any service | Cache diverges silently across 2+ Azure instances |
| Unbounded list query (no Skip/Take) | Returns all rows — crashes at 100k+ records |
| No `AsNoTracking()` on read queries | EF change tracker holds all returned objects in memory |
| Loading entity just to set one field | One extra SELECT per status change; use `ExecuteUpdateAsync` |
| `SaveChangesAsync` inside a loop | One DB round-trip per row; use `AddRange` + single save |
| Loop querying DB for related entities (N+1) | N SELECT statements instead of one `WHERE IN` |
| No `IsActive` filter on queries | Deactivated records appear in results |
| No composite partial index | Full-table scans at millions of rows |
| Hardcoded connection string | Fails in CI, staging, and Azure Key Vault environments |
| `context.Remove()` / hard delete | Violates the universal soft-delete contract |

## Self-Validation Checklist

Before completing any feature, run through every checkbox. Do not mark the feature done until all pass.

### Async / CancellationToken
- [ ] Every controller action has `CancellationToken ct` as the last parameter
- [ ] Every service method signature has `CancellationToken ct = default`
- [ ] Every repository method signature has `CancellationToken ct = default`
- [ ] No `.Result`, `.Wait()`, or `Task.Run()` anywhere in generated files
- [ ] Every EF method ends with `Async(ct)` — `ToListAsync(ct)`, `FirstOrDefaultAsync(ct)`, `SaveChangesAsync(ct)`, `AnyAsync(ct)`, `CountAsync(ct)`, `ExecuteUpdateAsync(..., ct)`

### EF Core Read Patterns
- [ ] Every list query has `.AsNoTracking()`
- [ ] Every query filters `x.IsActive`
- [ ] Every list query has `.Skip(skip).Take(take)`
- [ ] Results projected to DTOs — no raw entity returns from service methods

### Writes & Deletes
- [ ] Soft delete / deactivation uses `ExecuteUpdateAsync` — no load-then-mutate pattern
- [ ] Multi-row inserts use `AddRange` + single `SaveChangesAsync`
- [ ] Related entities resolved with batch `WHERE IN` query — no loop queries

### Entity & Schema
- [ ] Entity has `IsActive bool = true` (universal soft-delete flag)
- [ ] Entity has `UpdatedAt DateTime` updated on every write
- [ ] DbContext has composite partial index `HasFilter("\"IsActive\" = true")` for every query pattern
- [ ] High-growth entities (orders, visits, schedules) have partitioning TODO comment in migration

### Infrastructure
- [ ] Program.cs uses `AddDbContextPool` (not `AddDbContext`)
- [ ] Program.cs registers `AddRateLimiter` with global + auth sliding-window limiters
- [ ] All repositories and services registered as `Scoped` with a lifetime comment
- [ ] Connection string read via `GetConnectionString("DefaultConnection") ?? throw`
- [ ] Any caching uses `IDistributedCache` with cache-aside pattern — no `IMemoryCache`

#### 4. Generate Repository Implementation (`Repositories/{FeatureName}Repository.cs`)

**Rules:**
- `GetByIdAsync`: `FirstOrDefaultAsync` with `IsActive` guard — returns tracked entity for write callers
- `GetAllAsync`: `AsNoTracking()` + `.Where(x => x.IsActive)` + `Skip/Take` — never omit any of these
- `DeleteAsync`: `ExecuteUpdateAsync` — never load entity just to flip a flag
- Never call `SaveChangesAsync` inside Create/Update methods; call it only from `SaveChangesAsync()`
- Never use `context.Remove()` — soft delete via `IsActive = false` only

```csharp
using Microsoft.EntityFrameworkCore;
using sfa_api.Features.{FeatureName}.Entities;
using sfa_api.Infrastructure.Persistence;

namespace sfa_api.Features.{FeatureName}.Repositories;

public class {FeatureName}Repository(AppDbContext context) : I{FeatureName}Repository
{
    private readonly AppDbContext _context = context;

    // Tracked — callers that will update this entity afterward use this method
    public async Task<{FeatureName}?> GetByIdAsync(int id, CancellationToken ct = default)
        => await _context.{Entities}
            .FirstOrDefaultAsync(x => x.Id == id && x.IsActive, ct);

    public async Task<(IEnumerable<{FeatureName}>, int)> GetAllAsync(
        int skip, int take, string? search = null, CancellationToken ct = default)
    {
        var query = _context.{Entities}
            .Where(x => x.IsActive)    // always filter soft-deleted records
            .AsNoTracking();           // always on reads — no change-tracker overhead

        if (!string.IsNullOrWhiteSpace(search))
            query = query.Where(x => x.Name.ToLower().Contains(search.ToLower()));

        var total = await query.CountAsync(ct);
        var items = await query
            .OrderBy(x => x.Name)
            .Skip(skip)
            .Take(take)               // always paginate — never return unbounded list
            .ToListAsync(ct);

        return (items, total);
    }

    public async Task CreateAsync({FeatureName} entity, CancellationToken ct = default)
        => await _context.{Entities}.AddAsync(entity, ct);

    public Task UpdateAsync({FeatureName} entity, CancellationToken ct = default)
    {
        _context.{Entities}.Update(entity);
        return Task.CompletedTask;
    }

    // ExecuteUpdateAsync — direct SQL UPDATE; no entity load, no SaveChangesAsync needed after
    public async Task DeleteAsync(int id, CancellationToken ct = default)
        => await _context.{Entities}
            .Where(x => x.Id == id)
            .ExecuteUpdateAsync(s => s
                .SetProperty(x => x.IsActive,  false)
                .SetProperty(x => x.UpdatedAt, DateTime.UtcNow), ct);

    public async Task SaveChangesAsync(CancellationToken ct = default)
        => await _context.SaveChangesAsync(ct);
}
```

**Batch insert example (when creating multiple rows at once):**

```csharp
// CORRECT — one DB round-trip for N rows
_context.{Entities}.AddRange(entities);
await _context.SaveChangesAsync(ct);

// WRONG — N DB round-trips
foreach (var e in entities)
{
    _context.{Entities}.Add(e);
    await _context.SaveChangesAsync(ct); // one trip per row
}
```

**N+1 prevention (when resolving related entities):**

```csharp
// CORRECT — single WHERE IN query
var users = await _context.Users
    .Where(u => repIds.Contains(u.Id) && u.IsActive)
    .AsNoTracking()
    .ToListAsync(ct);

// WRONG — N queries inside a loop
foreach (var id in repIds)
    var user = await _repo.GetByIdAsync(id, ct); // one SELECT per ID
```

#### 9. Generate Service Implementation (`Services/{FeatureName}Service.cs`)

**Rules:**
- `IDistributedCache` for read-heavy data — **never `IMemoryCache`**
- Cache-aside pattern: check cache → miss → DB → populate cache
- Invalidate cache on every write
- Throw `NotFoundException`, `DuplicateResourceException`, `ValidationException`
- Set audit fields: `CreatedBy`, `UpdatedBy`, `UpdatedAt = DateTime.UtcNow`
- Call `SaveChangesAsync()` after Create/Update (not needed after `DeleteAsync` — uses `ExecuteUpdateAsync`)

```csharp
using Microsoft.Extensions.Caching.Distributed;
using System.Text.Json;
using sfa_api.Common.Errors;
using sfa_api.Features.{FeatureName}.DTOs;
using sfa_api.Features.{FeatureName}.Entities;
using sfa_api.Features.{FeatureName}.Repositories;
using sfa_api.Features.{FeatureName}.Requests;

namespace sfa_api.Features.{FeatureName}.Services;

public class {FeatureName}Service(
    I{FeatureName}Repository repo,
    IDistributedCache cache,               // IDistributedCache, NEVER IMemoryCache
    ILogger<{FeatureName}Service> logger) : I{FeatureName}Service
{
    private readonly I{FeatureName}Repository  _repo   = repo;
    private readonly IDistributedCache         _cache  = cache;
    private readonly ILogger<{FeatureName}Service> _logger = logger;

    public async Task<{FeatureName}Dto> GetByIdAsync(int id, CancellationToken ct = default)
    {
        // Cache-aside pattern
        // TTL guideline: reference data 1-24h | catalogues 15-60m | aggregates 5m
        //                approval/order status → NEVER cache
        var key    = $"{entities}:{id}";
        var cached = await _cache.GetStringAsync(key, ct);
        if (cached != null)
            return JsonSerializer.Deserialize<{FeatureName}Dto>(cached)!;

        var entity = await _repo.GetByIdAsync(id, ct)
            ?? throw new NotFoundException("{FeatureName}", id);

        var dto = MapToDto(entity);
        await _cache.SetStringAsync(key, JsonSerializer.Serialize(dto),
            new DistributedCacheEntryOptions
            {
                AbsoluteExpirationRelativeToNow = TimeSpan.FromHours(1)
            }, ct);

        return dto;
    }

    public async Task<{FeatureName}ListDto> GetAllAsync(
        int page, int pageSize, string? search = null, CancellationToken ct = default)
    {
        var skip = (page - 1) * pageSize;
        var (items, totalCount) = await _repo.GetAllAsync(skip, pageSize, search, ct);
        return new {FeatureName}ListDto(
            {Entities}: items.Select(MapToDto),
            TotalCount: totalCount,
            Page:       page,
            PageSize:   pageSize
        );
    }

    public async Task<{FeatureName}Dto> CreateAsync(
        Create{FeatureName}Request request, int? callerId, CancellationToken ct = default)
    {
        var entity = new {FeatureName}
        {
            // ... map request properties ...
            CreatedBy = callerId,
            UpdatedBy = callerId,
        };

        await _repo.CreateAsync(entity, ct);
        await _repo.SaveChangesAsync(ct);

        _logger.LogInformation("Created {FeatureName} {Id}", entity.Id);
        return MapToDto(entity);
    }

    public async Task<{FeatureName}Dto> UpdateAsync(
        int id, Update{FeatureName}Request request, int? callerId, CancellationToken ct = default)
    {
        var entity = await _repo.GetByIdAsync(id, ct)
            ?? throw new NotFoundException("{FeatureName}", id);

        // ... update properties ...
        entity.UpdatedBy = callerId;
        entity.UpdatedAt = DateTime.UtcNow;

        await _repo.UpdateAsync(entity, ct);
        await _repo.SaveChangesAsync(ct);

        // Invalidate on write
        await _cache.RemoveAsync($"{entities}:{id}", ct);
        return MapToDto(entity);
    }

    public async Task DeleteAsync(int id, CancellationToken ct = default)
    {
        // Verify exists before soft-deleting
        _ = await _repo.GetByIdAsync(id, ct)
            ?? throw new NotFoundException("{FeatureName}", id);

        // ExecuteUpdateAsync handles the SQL UPDATE — no SaveChangesAsync needed
        await _repo.DeleteAsync(id, ct);
        await _cache.RemoveAsync($"{entities}:{id}", ct);
    }

    private static {FeatureName}Dto MapToDto({FeatureName} entity) => new(
        Id:        entity.Id,
        // ... map properties ...
        IsActive:  entity.IsActive,
        CreatedAt: entity.CreatedAt,
        UpdatedAt: entity.UpdatedAt
    );
}
```

#### 10. Generate Controller (`Controllers/{FeatureName}sController.cs`)

**Rules:**
- `CancellationToken ct` on every action — ASP.NET Core injects it automatically
- Admin-only endpoints use `[Authorize(Roles = "Admin")]`
- Validate with FluentValidation before calling service

```csharp
using FluentValidation;
using Microsoft.AspNetCore.Authorization;
using Microsoft.AspNetCore.Mvc;
using System.Security.Claims;
using sfa_api.Common.Errors;
using sfa_api.Common.Extensions;
using sfa_api.Features.{FeatureName}.Requests;
using sfa_api.Features.{FeatureName}.Services;

namespace sfa_api.Features.{FeatureName}.Controllers;

[ApiController]
[Route("api/v1/{entities}")]
public class {FeatureName}sController(
    I{FeatureName}Service service,
    IValidator<Create{FeatureName}Request> createValidator,
    IValidator<Update{FeatureName}Request> updateValidator) : ControllerBase
{
    private readonly I{FeatureName}Service               _service         = service;
    private readonly IValidator<Create{FeatureName}Request> _createValidator = createValidator;
    private readonly IValidator<Update{FeatureName}Request> _updateValidator = updateValidator;

    [HttpGet]
    [Authorize]
    public async Task<IActionResult> GetAll(
        [FromQuery] int page = 1,
        [FromQuery] int pageSize = 10,
        [FromQuery] string? search = null,
        CancellationToken ct = default)
    {
        var correlationId = HttpContext.Items["CorrelationId"]?.ToString() ?? string.Empty;
        var result = await _service.GetAllAsync(page, pageSize, search, ct);
        return Ok(ResponseHelper.Ok(result, correlationId));
    }

    [HttpGet("{id}")]
    [Authorize]
    public async Task<IActionResult> GetById(int id, CancellationToken ct)
    {
        var correlationId = HttpContext.Items["CorrelationId"]?.ToString() ?? string.Empty;
        var result = await _service.GetByIdAsync(id, ct);
        return Ok(ResponseHelper.Ok(result, correlationId));
    }

    [HttpPost]
    [Authorize(Roles = "Admin")]
    public async Task<IActionResult> Create([FromBody] Create{FeatureName}Request request, CancellationToken ct)
    {
        var correlationId = HttpContext.Items["CorrelationId"]?.ToString() ?? string.Empty;

        var validation = await _createValidator.ValidateAsync(request, ct);
        if (!validation.IsValid)
            throw new Common.Errors.ValidationException(
                validation.Errors.ToDictionary(e => e.PropertyName, e => e.ErrorMessage));

        int.TryParse(User.FindFirstValue(ClaimTypes.NameIdentifier), out var callerId);
        var result = await _service.CreateAsync(request, callerId, ct);
        return CreatedAtAction(nameof(GetById), new { id = result.Id },
            ResponseHelper.Created(result, correlationId));
    }

    [HttpPut("{id}")]
    [Authorize(Roles = "Admin")]
    public async Task<IActionResult> Update(
        int id, [FromBody] Update{FeatureName}Request request, CancellationToken ct)
    {
        var correlationId = HttpContext.Items["CorrelationId"]?.ToString() ?? string.Empty;

        var validation = await _updateValidator.ValidateAsync(request, ct);
        if (!validation.IsValid)
            throw new Common.Errors.ValidationException(
                validation.Errors.ToDictionary(e => e.PropertyName, e => e.ErrorMessage));

        int.TryParse(User.FindFirstValue(ClaimTypes.NameIdentifier), out var callerId);
        var result = await _service.UpdateAsync(id, request, callerId, ct);
        return Ok(ResponseHelper.Ok(result, correlationId));
    }

    [HttpDelete("{id}")]
    [Authorize(Roles = "Admin")]
    public async Task<IActionResult> Delete(int id, CancellationToken ct)
    {
        await _service.DeleteAsync(id, ct);
        return NoContent();
    }
}
```

```csharp
var connectionString = builder.Configuration.GetConnectionString("DefaultConnection")
```

**Entity configuration in `OnModelCreating` — composite partial indexes required:**

```csharp
modelBuilder.Entity<{FeatureName}>(e =>
{
    e.HasKey(x => x.Id);
    e.Property(x => x.Id).UseIdentityColumn();

    // Unique constraint (add only if applicable)
    // e.HasIndex(x => x.Name).IsUnique();

    // Composite partial index — active rows only; CreatedAt DESC for sorted lists
    // Partial filter keeps index smaller at millions of soft-deleted rows
    e.HasIndex(x => new { x.IsActive, x.CreatedAt })
     .IsDescending(false, true)
```

## Hierarchical Features & Denormalization

The SFA API uses a 4-level geographic hierarchy:

```
Region → Area → Territory → Division
```

**Rule: each level stores all ancestor IDs as denormalized columns.**

| Entity    | Stores |
|-----------|--------|
| Region    | nothing above |
| Area      | `RegionId` (direct parent FK) |
| Territory | `AreaId` (direct FK) + `RegionId` (denormalized) |
| Division  | `TerritoryId` (direct FK) + `AreaId` + `RegionId` (denormalized) |

### Generating a Hierarchical Entity

**Entity:**
```csharp
public class Territory
{
    public int Id { get; set; }
    public string Name { get; set; } = string.Empty;
    public int AreaId   { get; set; }   // direct parent (FK)
    public int RegionId { get; set; }   // denormalized ancestor
    public bool IsActive { get; set; } = true;
    // ... audit fields ...
    public Area?   Area   { get; set; }
    public Region? Region { get; set; }
}
```

**Repository — `AsNoTracking` on reads; `IsActive` filter always:**
```csharp
// Tracked — for writes
public async Task<Territory?> GetByIdAsync(int id, CancellationToken ct = default)
    => await _context.Territories
        .Include(t => t.Area)
            .ThenInclude(a => a!.Region)
        .FirstOrDefaultAsync(t => t.Id == id && t.IsActive, ct);

// Dedicated method to resolve ancestors on write (tracked)
public async Task<Area?> GetAreaWithRegionAsync(int areaId, CancellationToken ct = default)
    => await _context.Areas
        .Include(a => a.Region)
        .FirstOrDefaultAsync(a => a.Id == areaId && a.IsActive, ct);

// Read-only list
public async Task<(IEnumerable<Territory>, int)> GetAllAsync(
    int skip, int take, string? search = null, CancellationToken ct = default)
{
    var query = _context.Territories
        .Where(t => t.IsActive)
        .AsNoTracking();

    if (!string.IsNullOrWhiteSpace(search))
        query = query.Where(t => t.Name.ToLower().Contains(search.ToLower()));

    var total = await query.CountAsync(ct);
    var items = await query
        .OrderBy(t => t.Name)
        .Skip(skip).Take(take)
        .ToListAsync(ct);

    return (items, total);
}
```

**Service — resolve the full ancestor chain from the parent on create/update:**
```csharp
var area = await _repo.GetAreaWithRegionAsync(request.AreaId, ct)
    ?? throw new NotFoundException("Area", request.AreaId);

var territory = new Territory
{
    Name     = request.Name,
    AreaId   = area.Id,
    RegionId = area.RegionId,  // already denormalized on Area — no extra query
};
```

**DbContext — composite partial index per FK:**
```csharp
modelBuilder.Entity<Territory>(e =>
{
    e.HasIndex(x => new { x.AreaId,   x.CreatedAt })
     .HasFilter("\"IsActive\" = true")
     .HasDatabaseName("idx_territories_area_created");

    e.HasIndex(x => new { x.RegionId, x.CreatedAt })
     .HasFilter("\"IsActive\" = true")
     .HasDatabaseName("idx_territories_region_created");

    e.HasOne(x => x.Area).WithMany().HasForeignKey(x => x.AreaId).IsRequired();
    e.HasOne(x => x.Region).WithMany().HasForeignKey(x => x.RegionId).IsRequired();
});
```

**DTO — include both ID and Name for every ancestor:**

## Key Principles

### Naming Conventions
- **Entities**: Singular PascalCase (`User`, `Product`, `Order`)
- **Controllers**: Plural with "Controller" suffix (`UsersController`, `ProductsController`)
- **Routes**: Lowercase plural (`/api/v1/users`, `/api/v1/products`)
- **Files**: Match class names exactly

### Soft Delete / Deactivation Pattern (CRITICAL)
- **NEVER hard-delete** — no `context.Remove()`, no `Remove(entity)`
- **`IsActive`** is the universal status flag — `true` = active, `false` = deactivated/deleted
- **Deactivation:** set `IsActive = false` via `ExecuteUpdateAsync`
- **Active-only queries:** always filter `.Where(x => x.IsActive)` — no global query filter
- **Do NOT add** `HasQueryFilter` in DbContext — filter explicitly per query

### DI Lifetime Rules
- Repositories: `Scoped` (shares request's DbContext)
- Services: `Scoped`
- HttpClient factories: `Singleton`
- Redis lock service: `Singleton`
- **Never register Scoped inside Singleton** — captive dependency causes stale data

### Audit Trail
- `CreatedBy`/`UpdatedBy` — set from caller ID
- `UpdatedAt = DateTime.UtcNow` — update on every modification
- `CreatedAt` — set once on creation

### Caching Guidelines
| Data type | TTL |
|-----------|-----|
| Geo hierarchy, config | 1–24 hours |
| Product catalogue / prices | 15–60 minutes |
| Rep's outlet list for today | 30–60 minutes |
| Dashboard counts / aggregates | 5 minutes |
| Approval status, order status, financial | **NEVER cache** |

### Error Handling
- Let `GlobalExceptionMiddleware` handle exceptions
- No try-catch in controllers/services unless recovering
- Throw domain exceptions: `NotFoundException`, `DuplicateResourceException`, `ValidationException`
- Validation exceptions must include field-level error dictionary
