Critical — launch blockers

1. Live production secrets committed to git Code fixed · rotation pending

Neon PostgreSQL password + JWT signing key in appsettings.json:10,15; Upstash token in appsettings.Development.json:8-9. The never-expiring admin token in CLAUDE.md is signed with that key = permanent backdoor. Confirmed by two independent advisors.

Done in code: secrets blanked in all committed files; fail-fast startup guard added (won't boot with an empty/weak JWT key or connection string); test suite decoupled to a dedicated test-only key; backdoor token removed from CLAUDE.md.

Still required (manual/ops): rotate the Neon password, JWT key & Upstash token in their consoles; purge git history; set the new values as env vars / user-secrets. The leaked values stay live until rotated.

2. Stock double-deduct via optional idempotency key ✓ Fixed

Billings dedupe is correct but keyed on the optional X-Idempotency-Key header (BillingsController.cs:261). Unique index is WHERE ClientBillId IS NOT NULL. Missing header → NULL id, no collision, DeductStockAsync runs twice. Stock deducted twice, no error.

Fixed: X-Idempotency-Key now mandatory on POST /api/v1/billings (400 if absent); added a regression test.

3. Seven migrations stuck behind data-dependent unique indexes ✓ Fixed

Partial UNIQUE indexes over active rows (Routes 20260615113307, Fleets/NotBillings 20260616062018) throw on duplicate active data. No cleanup migration, no migrate-on-startup guard → deploy boots green then 500s on missing columns.

Fixed: prepended idempotent dedup SQL to those migrations' Up() (keep newest per group, deactivate/soft-delete older dups) that runs before each CREATE UNIQUE INDEX — verified via ef migrations script. Added a startup schema guard: the app now refuses to boot with pending migrations (Npgsql + non-Dev only) instead of 500-ing later.

Still required (ops): apply migrations as an explicit deploy step (ef database update / bundle) — now enforced by the guard. Review the auto-deactivated duplicate rows after first apply.

High

4. Bill grand total can go negative ✓ Fixed

Validator ignored the bill discount subtracted alongside returns. Added a service-level guard on the actual rounded figures: rejects with 422 BILL_TOTAL_NEGATIVE before a billing number is burned. Added a regression test.

5. Cancelled/rejected bills counted as revenue ✓ Fixed

Outlet summary summed all bill states; now sums only revenue bills (IsRevenueBill: not Cancelled, not Rejected) while still listing cancelled ones. Reconciles with monthly totals.

6. NotBillings advertises idempotency it never wires up ✓ Fixed

Mirrored the Billings pattern: ClientRecordId column + filtered-unique index, fast-path + replay-catch, header now mandatory. Added service tests (feature had zero coverage).

7. Transactional entities have no concurrency token ✓ Fixed

Added xmin RowVersion to Billing/SalesInvoice/GRN/PurchaseOrder + a DbUpdateConcurrencyException → 409 mapping. A status transition can no longer double-execute if the lock expires. (Stock already row-locked; ledger append-only.)

8. Login rate-limit bypassable ✓ Fixed

Limiter now partitions on framework-resolved RemoteIpAddress, not raw X-Forwarded-For; forwarded headers trusted only from configured known proxies.

9. Deactivated users keep access ≤7 days ✓ Fixed

AuthService.RefreshAsync now checks IsActive and revokes the token family on a deactivated user — no more minting tokens after termination.

10. Background jobs run per-instance with no lock ✓ Fixed

All four BackgroundServices now acquire a distributed lock per tick (non-blocking; skip if another replica holds it).
